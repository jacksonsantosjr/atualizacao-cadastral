## 🎭 1. OS ESPECIALISTAS (Agents)
Estes são os seus "pilotos". Cada um possui uma especialidade profunda e deve ser invocado com a menção `@nome-do-agente`.

| Especialista | O que ele faz (Domínio) | Quando Invocar (Exemplo) |
| :--- | :--- | :--- |
| **`@orchestrator`** | **O Maestro.** Coordena múltiplos agentes em paralelo para tarefas complexas. | *"@orchestrator, construa um sistema de chat com backend Python e frontend React."* |
| **`@project-planner`** | **O Arquiteto.** Cria planos detalhados (`PLAN.md`), define estruturas de arquivos e tarefas. | *"@project-planner, planeje a estrutura de pastas para um novo app de CRM."* |
| **`@frontend-specialist`** | **O Artista Digital.** Focado em React, Next.js, UI/UX e performance de interface. | *"@frontend-specialist, crie um componente de dashboard com Glassmorphism."* |
| **`@backend-specialist`** | **O Motor.** Especialista em Node.js, Python, APIs REST/GraphQL e segurança de servidor. | *"@backend-specialist, implemente um endpoint de autenticação JWT seguro."* |
| **`@database-architect`** | **O Guardião dos Dados.** Desenha esquemas SQL/NoSQL, otimiza queries e migrações. | *"@database-architect, otimize esta query que está demorando 2 segundos."* |
| **`@security-auditor`** | **O Escudo.** Auditoria completa de segurança (OWASP), busca vulnerabilidades e falhas. | *"@security-auditor, analise este código em busca de falhas de XSS ou Injeção."* |
| **`@debugger`** | **O Detetive.** Especialista em análise de causa raiz e correção de bugs complexos. | *"@debugger, a tabela de amortização está pulando datas. Encontre o erro."* |
| **`@devops-engineer`** | **O Operador.** Cuida de Docker, CI/CD, deploys no Render/Vercel e monitoramento. | *"@devops-engineer, configure o arquivo Dockerfile para produção."* |
| **`@test-engineer`** | **O Validador.** Escreve testes unitários, integração e garante cobertura total de código. | *"@test-engineer, escreva testes Jest para todos os hooks da página inicial."* |
| **`@seo-specialist`** | **O Consultor de Visibilidade.** Otimiza Meta tags, Core Web Vitals e estratégias de busca. | *"@seo-specialist, faça auditoria de SEO na landing page para melhorar o ranking."* |

## 🛠️ 2. AS HABILIDADES (Skills)
As Skills são módulos de conhecimento profundo que os agentes carregam sob demanda. Você pode pedir para o agente usar uma skill específica se quiser um resultado focado.

### 🎨 Design & Frontend
*   **`ui-ux-pro-max`**: A inteligência suprema de design. Possui **50 estilos visuais**, **95 paletas de cores** e **50 tipografias premium**.
    *   *Uso:* "Crie a tela usando o estilo `Glassmorphism Modern` da skill ui-ux-pro-max."
*   **`frontend-design`**: Focada em heurísticas de usabilidade e padrões de UI modernos.
*   **`tailwind-patterns`**: Padrões avançados para Tailwind CSS v4.

### ⚙️ Backend & Infra structure
*   **`api-patterns`**: Domínio total de arquiteturas REST, GraphQL e tRPC.
*   **`app-builder`**: O motor de scaffolding. Contém **12 templates** completos para diversos tech stacks.
*   **`deployment-procedures`**: Protocolos seguros de deploy e estratégias de rollback.

### 🔍 Qualidade & Segurança
*   **`vulnerability-scanner`**: Auditoria profunda baseada no OWASP 2025.
*   **`systematic-debugging`**: Metodologia de 4 fases para resolver qualquer bug.
*   **`tdd-workflow`**: Implementação rigorosa de Ciclo Vermelho-Verde-Refatoração.

## ⚡ 3. OS WORKFLOWS (Slash Commands)
Estes são "atalhos de processo" que acionam rotinas automáticas de alta complexidade.

| Comando | O que faz | Exemplo Prático |
| :--- | :--- | :--- |
| **`/brainstorm`** | Ativa o modo de descoberta via perguntas socráticas para entender o projeto. | `/brainstorm, quero criar um app de gestão para minha barbearia.` |
| **`/plan`** | Cria o arquivo `PLAN.md` com todo o cronograma e estrutura necessária. | `/plan, crie o plano de implementação para o novo módulo de faturamento.` |
| **`/orchestrate`** | Inicia a execução paralela de múltiplos especialistas (mínimo 3 agentes). | `/orchestrate, implemente o sistema de login com UI Glassmorphism.` |
| **`/debug`** | Abre um processo de investigação técnica para encontrar e fixar bugs "impossíveis". | `/debug, por que o servidor cai sempre que subo um PDF grande?` |
| **`/ui-ux-pro-max`** | Ativa o gerador de interfaces premium com base no manual de 50 estilos. | `/ui-ux-pro-max, crie uma landing page no estilo Neumorphism Dark.` |
| **`/deploy`** | Valida a segurança e executa o deploy oficial no ambiente de produção. | `/deploy, suba a versão v1.2 para o Render.` |
| **`/status`** | Gera um quadro de status completo do projeto, tarefas pendentes e blockers. | `/status, como está o progresso do desenvolvimento atual?` |

## 🤖 4. COMO INVOCAR O PODER MÁXIMO
Para obter o melhor dos seus agentes, siga o **Protocolo Antigravity**:

1.  **Contexto é Tudo:** Não peça apenas "um botão". Peça: *"@frontend-specialist, crie um botão de Call to Action usando a skill `frontend-design` e o estilo `Glassmorphism`."*
2.  **O Fluxo Ideal:**
    *   Comece com `/brainstorm` para alinhar as ideias.
    *   Use `/plan` para estruturar o trabalho.
    *   Chame `/orchestrate` para deixar os agentes trabalharem em paralelo.
    *   Finalize sempre com os scripts de validação:
        *   `python .agent/scripts/checklist.py .` (Para checagens rápidas de lint e segurança).
        *   `python .agent/scripts/verify_all.py .` (Para validação completa pré-deploy).
